# Glossary

## A

**Action** - A ROS 2 communication pattern for long-running tasks that provide feedback and can be canceled, consisting of goal, feedback, and result messages.

**Articulation Body** - Unity component that simulates joint constraints and rigid body dynamics for articulated objects like robot arms.

## B

**Behavior Tree** - A hierarchical structure used in robotics and AI for organizing complex behaviors and decision-making processes.

**Bridge** - A software component that connects two different systems or communication protocols, such as ROS and Unity.

## C

**Coordinate Frame** - A reference system used to describe the position and orientation of objects in space, typically defined by x, y, z axes and rotation.

**Control Theory** - The mathematical study of dynamical systems with inputs, and how their behavior is modified by feedback.

## D

**Deep Learning** - A subset of machine learning based on artificial neural networks with representation learning.

**Differential Drive** - A common mobile robot configuration using two independently controlled wheels on an axle.

**Digital Twin** - A virtual replica of a physical system that simulates its behavior and characteristics.

**Domain Randomization** - A technique for training AI models on randomized simulation environments to improve real-world transfer.

## E

**Embodiment** - The physical instantiation of an AI system, allowing it to interact with the physical world.

**End Effector** - The device at the end of a robot arm designed to interact with the environment (e.g., gripper, tool).

## F

**Forward Kinematics** - The process of calculating the position and orientation of a robot's end-effector given the joint angles.

**Frame** - A coordinate system in ROS used to describe positions and orientations of objects relative to each other.

## G

**Gazebo** - A physics-based 3D simulator for robotics that provides accurate simulation of robots in complex environments.

**Geometric Jacobian** - A matrix that relates joint velocities to end-effector velocities in robotics.

**Gripping Force** - The force applied by a robot's gripper to hold an object securely.

## H

**Hardware Abstraction Layer (HAL)** - Software layer that allows higher-level software to interact with hardware without knowing hardware-specific details.

**Human-Robot Interaction (HRI)** - The study of interactions between humans and robots.

## I

**Inverse Kinematics** - The process of calculating the joint angles needed to achieve a desired position and orientation of a robot's end-effector.

**IMU (Inertial Measurement Unit)** - A device that measures and reports a body's specific force, angular rate, and sometimes the magnetic field surrounding the body.

**Inertial Frame** - A reference frame in which Newton's laws of motion hold true without the need for fictitious forces.

## J

**Joint** - A connection between two links in a robot that allows relative motion between them.

**Joint Space** - The space defined by all possible joint angle configurations of a robot.

## K

**Kinematics** - The study of motion without considering the forces that cause the motion.

**Kinetic Chain** - A system of rigid bodies connected by joints that transmits motion.

## L

**LiDAR (Light Detection and Ranging)** - A sensing method that uses light in the form of a pulsed laser to measure distances.

**Localization** - The process of determining the position and orientation of a robot within its environment.

**Logistics** - The planning, implementation, and control of the efficient movement and storage of goods.

## M

**Middleware** - Software that provides common services and capabilities to applications beyond what's offered by the operating system.

**Mobile Robot** - A robot that can move around in its environment using actuated wheels, tracks, or legs.

**Monte Carlo Localization (MCL)** - A probabilistic algorithm for robot localization using particle filters.

## N

**Navigation** - The process of planning and executing a path for a robot to move from one location to another.

**Neural Network** - A computing system inspired by biological neural networks that constitute animal brains.

**Node** - A process that performs computation in ROS (Robot Operating System).

## O

**Odometry** - The use of data from motion sensors to estimate change in position over time.

**Operational Space** - The space in which the end-effector of a robot operates (position and orientation in Cartesian space).

## P

**Path Planning** - The computational process of finding a valid path from a start to a goal state.

**Perception** - The process of interpreting sensory information to understand the environment.

**Physics Engine** - Software that simulates physical systems, including collisions, forces, and motion.

**Point Cloud** - A set of data points in space, representing the external surface of an object or environment.

**Prismatic Joint** - A joint that provides linear sliding movement between two links.

**Publisher** - A ROS entity that sends messages to a topic.

## Q

**Quaternion** - A mathematical construct used to represent rotations in 3D space, avoiding gimbal lock.

## R

**Revolute Joint** - A joint that provides rotational movement between two links.

**Robot Operating System (ROS)** - Flexible framework for writing robot software, providing operating system-like functionality on a heterogeneous computer cluster.

**ROS 2** - The second generation of the Robot Operating System with improved architecture for real-world applications.

**Rigid Body** - An idealization of a solid body in which deformation is neglected.

## S

**Sensor Fusion** - The process of combining sensory data from multiple sources to improve accuracy and reliability.

**Simulation** - The imitation of the operation of a real-world process or system over time.

**SLAM (Simultaneous Localization and Mapping)** - The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it.

**Subscriber** - A ROS entity that receives messages from a topic.

**System Integration** - The process of bringing together the component subsystems into one system and ensuring that the subsystems function together.

## T

**Topic** - A named bus over which nodes exchange messages in ROS.

**Trajectory** - A time-ordered sequence of position and orientation states for a robot or object.

**Twist** - A ROS message type containing linear and angular velocities.

## U

**URDF (Unified Robot Description Format)** - An XML format for representing a robot model including visual, collision, and kinematic properties.

**Unity** - A cross-platform game engine used for creating 3D simulations and visualizations.

**Unity Visualization** - Using the Unity game engine to create visual representations of robot systems and environments.

## V

**V-SLAM (Visual Simultaneous Localization and Mapping)** - SLAM using visual sensors such as cameras.

**VLA (Vision-Language-Action)** - A paradigm integrating visual perception, language understanding, and physical action in AI systems.

**Velocity** - The rate of change of displacement with respect to time, including both speed and direction.

## W

**Waypoint** - A reference point on a path used with a navigation system.

**Wheel Odometry** - Estimation of robot position based on wheel rotation measurements.

## X, Y, Z

**XYZ Coordinates** - A three-dimensional coordinate system using X, Y, and Z axes to define positions in space.
